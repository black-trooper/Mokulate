Mokulate（ZMK）では、キーマップを変更する方法が大きく分けて **3 種類** あります。  
それぞれ使い勝手や必要な作業が異なるため、目的に合わせて選んでください。


| 方法 | 難易度 | 再書込み | カスタマイズ性 | 備考 |
|------|--------|-------------------|----------------|------|
| **ZMK Studio** | ★☆☆ | 不要 | △ 基本機能 | 最も簡単 |
| **Keymap Editor** | ★★☆ | 必要 | ○ 中級向け | GitHub が必要 |
| **ファーム書換** | ★★★ | 必要 | ◎ 上級向け | 全機能にアクセス可能 |


## 1. ZMK Studio

**最も簡単で、初心者におすすめの方法です。**

ZMK Studio はブラウザ上で動作する ZMK の公式ツールで、キーボードのレイアウトを見ながら直感的にキーマップを変更できます。

### 特徴

- ブラウザで動く
- キーをクリックするだけで割り当て変更
- 変更後は「Apply」するだけで即座に反映される（再書き込み不要）
- プロファイルごとの切り替えも簡単
- レイアウト表示で編集しやすい

### 簡易手順

1. [ZMK Studio](https://zmk.studio/) を開く
2. Mokulate の右側を USB で接続する
3. 編集したいキーをクリック
4. 割り当てたいキーを選択
5. 「Apply」を押して保存  
   （※ファームウェアの再書き込みは不要）

ZMK Studio が使える環境であれば、この方法が最も手軽です。

---

## 2. Keymap Editor

**GitHub リポジトリを作る必要がありますが、プログラミングなしで  
ZMK Studio よりも複雑なカスタマイズができます。**

Keymap Editor は ZMK Config を *ブラウザ上で視覚的に編集するツール* です。  
GitHub Actions と連携し、自動ビルドされたファームウェアが生成されます。

### 特徴

- 自分の **zmk-config レポジトリ** が必要  
- キー配置やレイヤーを GUI で編集できる
- ZMK Studio より高機能（レイヤー管理・モディファイア設定など）
- GitHub Actions により UF2 を自動生成
- プログラミング不要だが、環境整備と GitHub 操作は必要

### 備考

詳細な操作方法は **自分で調べて頑張ってください。**

---

## 3. ファームウェアを書き換える

ZMK Config リポジトリ内の `.keymap` / `.overlay` ファイルを **直接テキスト編集してビルドする方法** です。

### 特徴

- 最も柔軟で細かいカスタマイズが可能  
  （レイヤー、動作ロジック、モジュール設定など）
- GitHub Actions が新しい UF2 を自動ビルド
- キーごとの動作を ZMK の記法で完全に制御できる

### 備考

構文や設定など覚えることが多く、**高度なカスタマイズ向けです。**  
詳細は自分で調べてください。
